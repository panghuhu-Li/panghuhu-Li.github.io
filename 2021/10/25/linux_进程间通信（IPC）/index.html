Linux 进程间通信（IPC）

发表于 2021-10-25 | 分类于 linux 东北大学石凯老师 |

进程间通信（IPC）

进程间通信（interprocess communication，缩写为IPC）是只在某一个操作系统之上的不同进程间各种数据传递的方式。进程间通信分为直接通信和间接通信（间接通信需要一个共享的mailbox来通信，如共享内存、文件等）。显然，间接通信涉及了“共享资源”，必然要考虑同步问题。直接通信包括：管道、System V消息队列、Posix消息队列、远程过程调用（remote procedure call, 缩写为RPC，Java的RMI就是一种RPC）等。另外socket也可以看作是一种直接通信（因为socket也可以使用本地连接）。间接通信包括：共享内存、共享文件等。

进程间通信是一个非常宏大而复杂的问题，Richard Stevens的名著“Unix Network Programming, Voloume 2: Interprocess Communications”全书就是讲解IPC这一个问题。之前我们已经讲解过管道和socket，这部分我们选择一些更易于理解的IPC及同步机制进行讲解，比如我们选择System V共享内存而不是POSIX共享内存，选择POSIX信号量而不是System V信号量。

System V共享内存

共享内存是指多个进程将同一个共享内存区映射到它们各自的逻辑地址空间。采用共享内存通信的好处是效率高：因为共享内存区分配在内核之外，进程可以直接读写内存，而不需要切换到内核，以及任何用户空间和内核空间的数据的拷贝。而管道和消息队列等方式，则需要数据在用户空间和内核空间进行拷贝。请大家回忆一下，管道的底部是位于内核空间的。

使用共享内容需要两个步骤：

  1. 创建或找到一个共享内存段。显然，系统可以建立多个共享内存段。为了让所有需要通信的进程找到的是同一个共享内存段，我们需要给每个共享内存段设置一个独一无二的标识符。
  2. 将已经找到的共享内存段映射到本进程的逻辑地址空间。

进程同步：POSIX信号量

信号量Dijkstra发明的一种强大却又原始的同步工具。信号量本质上就是一个整数，只接受两种操作。wait()操作会对信号量减一，当信号量为0时wait()操作就会被阻塞，直到信号大于0时才能减一；signal()操作会对信号量无条件加一。

一个初始为1的信号量可以用于互斥。

例：mutexwrite，用于互斥写共享内存

sem_open(“mutexsem”, O_CREAT, 0644, 1)会创建和初始化一个信号量，该信号量会对应于/dev/shm的一个“sem.信号量名”的文件，在本例即“sem.mutexsem”。O_CREAT表示如果该信号量不存在，则按照第三个参数——mode，和第四个参数——信号量的初始值创建这个信号量。但如果该信号量存在，mode和初始值都会被忽略。请通过man sem_open了解详细细节。

创建成功后，我们可以在/dev/shm看到该信号量对应的文件。
